# AI Generation System Documentation

> **Last updated**: 2026-05-30
> **Database**: All media stored in `data/media/media.db.sqlite` (`media_assets` table)

## Overview

PipelineGen integrates Google Flow, NVIDIA Flux, and Ollama to generate images enriched with
LLM-generated semantic metadata. Assets are deduplicated, indexed for hybrid search, and uploaded to Google Drive.

- **Deduplication**: SHA-256 hashing to avoid duplicate files
- **Database**: `media.db.sqlite` (`media_assets` table) — unified for images, videos, audio, voiceovers
- **Google Drive**: Automatic upload with `metadata.json` sidecar
- **Semantic Enrichment**: LLM-generated `concept_tags`, `visual_objects`, `emotional_tone`, `search_text_expanded` via Ollama at ingest time (not at search time)

## Image Generation

### Models Available
- `FLUX.1-schnell` — fast generation (default, via NVIDIA cloud)
- `flux-1-dev` — highest quality (Black Forest Labs via NVIDIA cloud)
- `google-flow` — Google Imagen via google-accounting Python bridge

### API Endpoint: `POST /api/images/generate/nvidia`

```json
{
  "prompt": "A futuristic city at sunset, cinematic lighting",
  "model": "FLUX.1-schnell",
  "width": 1280,
  "height": 720
}
```

### What happens at ingest
1. Image generated by selected model
2. SHA-256 hash computed, duplicates rejected
3. Image saved to `data/images/<prompt-slug>/<hash>.png`
4. `scripts/semantic_tagger.py` called with prompt + style:
   - **Taxonomy match** (YAML): subjects, tags, categories, mood
   - **YAKE/spaCy**: dynamic keyword extraction
   - **Ollama** (`gemma3:4b`): generates `concept_tags`, `visual_objects`, `emotional_tone`
   - Builds `search_text_expanded` blob (used for FTS5/BM25 at query time — zero LLM)
5. `metadata.json` sidecar uploaded to Google Drive alongside the image
6. Record created in `media.db.sqlite` (`media_assets` table)

## Script-to-Images Pipeline

### API Endpoint: `POST /api/script/generate-from-source`

Generates a full video script from source text, creates N image variants per scene,
enriches metadata via Ollama, and exports to a Google Doc.

**Parallel Execution & Performance:**
The image generation phase for scripts has been significantly optimized. PipelineGen now uses a concurrent worker pool to generate up to **6 scenes simultaneously**.
When using the Google Vids generator (`google-accounting` backend), the Python service utilizes a Playwright session pool (`MAX_WARM_CONTEXTS = 6`) to maintain multiple pre-warmed browser instances per account. This eliminates browser startup overhead and allows all 6 Go workers to execute browser automation flows in parallel, drastically reducing the total generation time for multi-scene scripts.

```json
{
  "source_text": "...",
  "language": "en",
  "style": "whiteboard",
  "visual_style": "whiteboard",
  "images_per_scene": 3,
  "width": 1024,
  "height": 1024,
  "title": "My Video"
}
```

**`images_per_scene`** (default 1, max 5): generates N visual variants per scene with different
angles (primary, close-up, wide shot, side view, overhead) to avoid YouTube content-reuse flags.

Response JSON per scene:
```json
{
  "text": "scene text",
  "image": { "drive_link": "...", "local_path": "..." },
  "image_links": ["link1", "link2", "link3"]
}
```

## Video Animation (Zoom-Out)

### API Endpoint: `POST /api/images/animate`

```json
{
  "image_hash": "a1b2c3d4...",
  "duration": 7
}
```

Generates a 7-second 1920×1080 H.264 zoom-out MP4 via `scripts/animate_image.py` (FFmpeg).
Output saved to `data/animations/` and uploaded to Drive.

## Semantic Search Architecture

Images are searchable via **hybrid BM25 + vector search** with zero LLM at query time:

| Field | Source | Used for |
|-------|--------|---------|
| `search_text` | Taxonomy + YAKE | Basic FTS |
| `search_text_expanded` | LLM (Ollama, at ingest) | Full hybrid FTS blob |
| `concept_tags` | LLM (Ollama, at ingest) | Synonym/concept matching |
| `visual_objects` | LLM (Ollama, at ingest) | Object-level matching |
| `emotional_tone` | LLM (Ollama, at ingest) | Intent/tone matching |
| `embedding_json` | Sentence-Transformers | Vector cosine similarity |
